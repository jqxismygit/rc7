### 整体思路（先说结论）

**目标**：在现有代码基础上，严格对齐 `task.md.resolved` 的阶段规划，用 `apps/Prd/小程序.MD` 作为功能与交互“单一真相来源”，把小程序前端补完成一个：  
- 有统一“暗黑金属运动格”视觉、  
- 有完整业务闭环（登录 → 首页 → 购票/活动/世界杯 → 票夹/我的 → 发票/核销）、  
- 完整接入 CMS 与微信能力的生产级小程序。

下面我按“全局 & 模块”给你一个**可以直接开工的实现清单**，你可以按阶段勾任务。

### 一、全局设计系统 & 状态（对应阶段一）

- **全局设计系统（`uni.scss`）**
  - **主题方向**：暗黑金属运动风（以当前代码实现为准）  
    - 背景：`$cr7-black (#0D0D0D)` / `$cr7-dark (#1A1A1A)`，深色运动质感底
    - 卡片：`$cr7-card (#242424)` 搭配 `$cr7-border (#333333)` 描边与分割线
    - 主品牌色：荧光黄绿 `$cr7-gold (#D8FC0F)`，用于主按钮、徽章高光、关键状态
    - 金色层次：`$cr7-gold-light (#E5FE4D)` / `$cr7-gold-dark (#A8C00C)` 作为文字与渐变层
    - 渐变：`$gradient-gold`（金色渐变按钮、徽章）、`$gradient-dark` / `$gradient-hero`（暗背景与大片遮罩）
    - 状态色：`$cr7-red (#D9001B)`、`$cr7-success (#2ECC71)`、`$cr7-warning (#F39C12)`
  - **通用设计变量建议**（沿用 `uni.scss` 现有体系）：  
    - 色彩：`$cr7-*` 系列 + `$text-*` 系列统一管理文字与背景
    - 圆角：`$radius-sm` / `$radius-md` / `$radius-lg` / `$radius-xl` / `$radius-round`
    - 字体：`$font-xs` ~ `$font-hero` 分级控制标题与正文
    - 阴影：`$shadow-gold` / `$shadow-dark` / `$shadow-card`，统一按钮、卡片的立体感
    - 通用类：`.btn-gold`、`.btn-outline-gold`、`.card-dark`、`.tag-active` / `.tag-ended` / `.tag-countdown` 等在各页面复用
- **页面结构（`pages.json`）分层**
  - `tabBar`：`/pages/index/index`, `/pages/my-tickets/my-tickets`, `/pages/profile/profile`
  - 非 tab：`/pages/login/login`, `/pages/messages/messages`, `/pages/ticket-purchase/ticket-purchase`, `/pages/event-detail/event-detail`, `/pages/worldcup/worldcup`, `/pages/legal/terms`, `/pages/legal/privacy`, `/pages/profile/order-list`, `/pages/profile/invoice`, `/pages/schedule/schedule`, `/pages/video/video-list`, `/pages/career/career`
- **全局状态管理（建议用 Pinia / Vuex，建 `store/index.ts`）**
  - `userStore`：
    - `profile`（头像、昵称、邮箱、手机号、角色：普通用户/员工）
    - `token`、`isLoggedIn`
  - `appStore`：
    - `locationCity`、`lbsAuthorized`、`cityList`
    - `unreadMessageCount`
    - `cmsConfig`（协议/隐私、首页 Banner & 信息流、品牌墙、活动、世界杯、行程、视频、职业生涯等）
  - `ticketStore`：
    - `currentOrderDraft`（购票过程中临时订单）
    - `myTickets`、`thirdPartyTickets`
  - **统一请求与拦截器**
    - 封装 `request`：自动带 token、错误码统一弹 Toast  
    - 未登录接口 → 跳转 `/pages/login/login`  
    - 权限接口（员工核销、开票等）根据用户角色拦截

### 二、登录与用户中心（阶段二）

#### 1. `pages/login/login.vue`

- **协议勾选 + CMS 拉取**
  - 首屏加载时调用 CMS 接口：`/cms/legal` → 返回 `termsUrl`, `privacyUrl` 或 HTML 内容。
  - 协议文案区域：勾选框默认未勾选（对齐 PRD），点击“用户协议”“隐私政策” → 跳转到 `terms.vue` / `privacy.vue`，用 `rich-text` 渲染。
  - 登录按钮 `disabled = !checkedAgreements`，点击时如未勾选再做一次校验 Toast。
- **一键登录 & 未注册自动注册**
  - 用微信提供手机号一键登录接口（后端统一为“登录或注册”API），前端只负责：
    - 成功 → 保存 `token` + `userProfile` 到 `userStore`，`switchTab` 回首页。
    - 失败（用户拒绝授权等）→ 弹窗并留在当前页。
- **返回主页**
  - 顶部或左上角提供“返回首页”按钮：`switchTab('/pages/index/index')`。

#### 2. `pages/profile/profile-edit.vue`

- **头像**
  - 三个入口：使用微信头像、相册选择、拍照；组件复用 uni 选择图片能力。
- **昵称**
  - 默认微信昵称，可手动编辑，`maxlength=16`，超长给红字提示。
- **邮箱**
  - 单独输入框，失焦时做简单正则校验。
- **退出登录**
  - 点击 → 二次确认弹窗 → 确认后清空 `userStore`，`reLaunch` 至首页。

#### 3. `pages/profile/profile.vue`

- **主信息区**
  - 默认头像 + 默认昵称（未设置走微信 UID），风格同暗黑金属（头像边框发光圈）。
- **功能列表**
  - `我的订单`、`发票开具`、`隐私政策`、`服务协议`、`发票抬头`、`联系客服`。
- **发票抬头跳转**
  - 调用微信原生发票抬头管理小程序 API（后端给路径/参数），封装为 `openWeChatInvoiceTitleApp()`。
- **微信客服跳转**
  - 调起微信客服对话窗口（小程序客服能力），在该入口统一封装 `openWeChatCustomerService()`。

### 三、首页 + 消息（阶段三）

#### 1. `pages/index/index.vue`

- **LBS 授权与城市池**
  - `onLaunch` 或首次进首页：
    - 若未授权 → 弹系统授权窗口，引导打开定位。
    - 成功后：获取城市 → 存入 `appStore.locationCity`，请求后端城市池，缓存。
  - 城市选择器：点击城市名 → 打开城市列表弹层，可切换城市（更新全局）。
- **消息红点联动**
  - 进入首页/消息返回时调用 `/messages/unread-count`，写入 `appStore.unreadMessageCount`。
  - 顶部消息 icon 使用角标 `v-if="unreadMessageCount > 0"`.
- **轮播与信息流富媒体**
  - Banner 数据从 CMS `/cms/home/banners` 获取。
  - 点击跳转 `信息流详情页`（可重用 `pages/index/index.vue` 下的详情弹层或独立页）。
  - 富媒体：后端返回结构化数据（文本块、图片、视频），前端用 `rich-text`+ 视频组件组合渲染；全屏视频直接 `requestFullScreen`。
- **C 罗专区、品牌墙**
  - 行程日历/视频集锦/职业生涯入口：路由直接跳对应页（下面单独说）。
  - 联名品牌 Logo 墙：横向可滚列表 + “查看更多”收尾；点击品牌卡：
    - 展示品牌故事详情（图文页）
    - 点击“进入官网”按钮 → 弹二次确认：“即将离开本小程序，前往第三方小程序” → 用户确认才跳转。

#### 2. `pages/messages/messages.vue`

- **列表 & 排序**
  - 接口按时间倒序返回（最新在前），前端确保按 `createdAt desc` 展示。
- **一键全部已读**
  - 仅当存在未读消息时按钮高亮，可点击；否则置灰。
  - 点击 → 调用批量已读接口 → 更新 `unreadMessageCount = 0`，所有 item 状态改为已读（灰色字体）。
- **一键清除**
  - 仅当存在任何消息时高亮；无消息时置灰。
  - 点击 → 二次确认弹窗 → 调接口删除已读消息 → 刷新列表。
- **交互防抖**
  - 执行批量操作期间：按钮置为 `loading`+禁用，避免连点。

### 四、购票流 & 活动 / 世界杯（阶段四）

#### 1. `pages/ticket-purchase/ticket-purchase.vue`

- **日期选择组件**
  - 引入自定义日历组件，支持：
    - 只允许可售日期可点（根据库存数据渲染置灰）。
- **票种与库存联动**
  - 券列表从后端获取，字段包括：价格、是否可退、可用时间段、剩余库存。
  - 当天售罄 → 卡片置灰 + 不可点击（仍展示信息）。
  - 选择票种 → 下方区域展示“是否可退 / 可用时段”标签。
- **数量与金额计算**
  - 数量组件限制 1–6 张。
  - 总价 = 单价 × 数量，实时更新；在 `currentOrderDraft` 中保存。
- **联系人手机号**
  - 默认使用用户当前登录手机号，可修改。
- **微信支付流程**
  - 点击“立即提交”：
    - 调后端“创建订单”接口 → 返回预支付信息。
    - 拉起微信支付。
    - 支付成功 → 后端将订单状态改为“待核销”，生成 9 位字母数字券码，写库；前端跳转到“订单确认/票夹”。
    - 用户取消支付 → 后端取消订单并不出现在“我的票券/订单”列表中；前端 Toast “已取消支付”并回到购票页或票夹。

#### 2. 线下活动（`pages/event-detail/event-detail.vue` + 列表）

- **瀑布流列表 + 懒加载**
  - 列表页接口分页，滚动触底加载下一页。
- **排序逻辑**
  - 请求参数中携带排序规则：先按活动开始时间（最近在上），再按状态优先级：
    - 进行中 > 倒计时 X 天 > 已结束。
- **报名详情页**
  - “查看详情”模式：不展示报名表单；  
  - “立即报名”模式：底部展示表单（姓名、身份证、手机号）。
- **前端校验**
  - 身份证 18 位 & 号段格式简单正则；手机号 11 位中国手机号正则。
- **报名/取消报名时间窗**
  - 根据活动数据的 “报名开始时间” 与 “活动开始时间”：
    - 只有在 `[报名开始, 活动开始前)` 时间段显示“报名/取消报名”按钮。
- **报名状态**
  - 已报名再进入详情 → 展示“取消报名”按钮；取消后可重新报名。

#### 3. 世界杯互动（`pages/worldcup/worldcup.vue`）

- **Tab 切换**
  - `tab = "schedule" | "ranking"`，顶部分段控制。
- **赛程**
  - 展示日期、主客队、时间、组别，当日/当段比赛。
  - 比赛开始前：用户可对每场做一次“胜/平/负”单选竞猜。
- **结果反馈**
  - 比赛结束后，后端返回用户竞猜结果字段：
    - 显示“已猜对 / 未竞猜 / 已猜错”标签，颜色区分。
- **积分榜**
  - 从 CMS/接口倒拉，以官方积分排序显示：排名、国家、组别、积分。

### 六、票夹 & 核销（阶段六）

#### 1. 票夹首页 `pages/my-tickets/my-tickets.vue`

- **排序与标签**
  - 按购买时间倒序（最新在最上）。
  - 卡片包含：缩略图、名称、张数、是否可退/条件、参观时间、票种标签（官方票/三方票）、状态（待核销/已完成/已退款）。
- **操作按钮可见性/禁用规则**
  - “申请退票”：仅支持官方票且满足“开展前 48 小时外”，否则置灰或隐藏。
  - “查看券码”：仅非退款中的订单可用。
  - “删除票券”：仅“已完成/已退款”可见，且二次确认，逻辑删除（列表隐藏、后端保留）。
- **工具条**
  - `票码兑换`、`三方票同步`、`发票开具`、`券码核销（员工）`
- **三方票同步**
  - 使用当前登录手机号向三方拉取订单；展示“三方票”标签。
  - 三方票禁止退票、禁止开票，仅展示。

#### 2. 用券/退券流

- **查看券码**
  - 未核销券：展示券码 + 二维码：
    - 二维码生成规则：券码 + 9 位随机数字。
    - 横向滑动展示同订单多券码。
  - 已核销券：相同界面但显著“已核销”标记。
  - 一键复制券码功能。
- **申请退票**
  - 符合条件时按钮可点，跳转退票确认页。
  - 展示订单号、实付金额、退款金额 → 仅原路退到微信。
  - 提交后 → “退款中” → 完成后更新为“已退款”；退票中/已退款禁止查看券码。
- **删除逻辑**
  - 二次确认弹窗；前端只做软删除（不再展示），后端逻辑删除标记。

#### 3. 员工核检

- “券码核销”按钮仅在员工角色时显示（或专用员工入口）。
- 调起后置摄像头，持续扫码：
  - 若票码+随机码匹配且在有效期且状态为待核销 → 核销成功，订单状态更新为已完成。
  - 若订单为退款中/已退款/已完成 → 核销失败并提示。
- 防重复核销：后端保证“一次即失效”，前端仅展示结果。

### 八、你可以直接开做的开发顺序建议

1. **先完成阶段一：**
   - 配好 `uni.scss` 主题、`pages.json` 路由分组、全局 `store` 与 `request` 封装。
2. **再完成阶段二 + 三：**
   - `login.vue` 全逻辑、`profile` 系列页面、`index.vue` LBS/红点/信息流、`messages.vue`。
3. **然后阶段四（购票 + 活动 + 世界杯）：**
   - 优先打通购票→支付→票夹这一条链，再丰富活动/世界杯交互。
4. **最后阶段五/六/七：**
   - 资料专栏（视频/职业/行程）、票夹细节（兑换/三方票/核销）、发票系统。
