 # CR7® LIFE 小程序交互 & UI 设计方案（基于 Axure 原型 V1.0）

 ## 1. 总体设计方向

 - **产品定位**：面向 C 罗球迷与泛体育人群的线下沉浸式博物馆 + 线上活动中枢，完成「选城 → 购票 → 入馆 → 活动/周边 → 留存复访」的完整闭环。
 - **基调**：深色背景 + 金属金点缀 + 少量 CR7 标志性红色，营造「球星更衣室 / 球场灯光」的氛围。
 - **核心记忆点**：登录页中的「CR7 圆形徽章」（Axure `注册登录.html` 中的圆形 + `CR7` 文案），作为整个小程序的品牌视觉锚点，在以下场景重复出现：
   - 登录页中央品牌展示
   - 首页顶部品牌角标
   - 一些空状态 / 结果页（如暂无票券、游戏未开放）
 - **交互风格**：动画克制但有力量，关键节点（登录完成、购票成功、票券核销成功）使用缩放 + 渐变 + 阴影的动效，日常浏览保持流畅干净。

 视觉与交互最终落地在 `apps/mini` 小程序内，以下方案以 Axure 输出的 HTML 为交互和布局的唯一参照，不受当前代码实现限制。

 ## 3. 核心用户路径设计

 ### 3.1 首次进入 & 授权登录

 参考 `注册登录.html`：

 - **进入流程**
   1. 用户从微信入口打开小程序 → 直接进入 `pages/login/login`。
   2. 背景为全屏 C 罗形象图；前景中间为 **CR7 圆形徽章** + 品牌标题「C 罗博物馆 CR7® LIFE」。
   3. 下方为「一键登录 / 注册」主按钮；底部有单选框文案「已阅读并同意《用户协议》《隐私政策》」。
 - **交互规则**
   - 未勾选单选框时主按钮置灰或不可点击，并在点击时给出轻微抖动 + 提示文案。
   - 点击《用户协议》《隐私政策》文字 → 打开对应长文页（`pages/legal/terms` / `pages/legal/privacy`），返回后保持勾选状态。
   - 勾选后，点击「一键登录 / 注册」：
     - 调用微信登录能力，获取 openid/unionid，后端完成注册或登录。
     - 登录成功 → 跳转 `pages/index/index`，并在首页顶部用 Toast 或轻微动效提示「欢迎来到 CR7® LIFE」。

 ### 3.2 首页路径

 参考 `首页.html`，首页主要区块：

 - **顶部区域**
   - 左侧：当前城市（例如「北京」）+ 向下箭头图标。
     - 点击城市 → 打开城市选择弹窗（列表 + 搜索），切换后刷新首页活动内容。
   - 右侧：消息图标 → 跳转 `pages/messages/messages`，显示系统通知、订单状态变更等。
   - 中间或右上角：小号 **CR7 圆形徽章** 作为品牌角标。
 - **Banner 区**
   - Swiper 轮播，多张活动/展馆宣传图，自动轮播 + 手势左右滑动。
   - 每张 Banner 可配置点击目标（购票页、线下活动详情、世界杯专题 H5）。
 - **热门活动区（Tab）**
   - Tab 标签：「购票」「线下活动」「世界杯」。
   - 不同 Tab 下展示对应列表：
     - 购票：当前城市可售展馆（图 + 馆名 + 时间 + 简要说明 + 状态标签）。
     - 线下活动：近期或城市限定活动（图 + 名称 + 时间 + 地址 + 状态：进行中/倒计时 N 天/已结束）。
     - 世界杯：与世界杯相关的互动/观赛活动。
   - 切换 Tab 时采用淡入动画，不刷新整页。
 - **C 罗专区**
   - 3 个入口卡片：「行程日历」「视频集锦」「职业生涯」。
   - 每个卡片包含小图标 + 标题 + 简短说明；点击跳转对应二级页。
 - **联名品牌**
   - 横向滚动品牌 logo 条（`scroll-view` 横向）。
   - 点击品牌 → 跳转 `pages/brands/brands` 品牌详情。

 ### 3.3 票夹路径

 参考 `票夹.html`：

 - **顶部工具栏**
   - 左侧按钮：「票码兑换」→ 跳 `pages/vote/vote`。
   - 右侧按钮：「三方票同步」→ 弹出说明弹窗，按说明进行第三方订单导入（后续可接真实接口）。
 - **票券列表**
   - 每张票券卡片包含：
     - 展馆名称（如「C 罗博物馆 - 中国北京馆」）。
     - 票种名称 + 数量（如「双人套票 2 人 × 6 张」）。
     - 参展时间。
     - 票种属性（官方票 / 三方票）。
     - 退票规则简述（如「开展前 48 小时可退」「特殊票种不可退」）。
     - 右上状态标签（`待核销` / `已退款` / 其他）。
   - 不同状态对应的操作按钮：
     - 待核销（官方票/三方票）：显示「查看券码」「申请退票」。
     - 已退款：可仅保留「查看券码」，或隐藏操作并显示「已退款」状态。
     - 特殊票不可退：不展示「申请退票」按钮。
 - **操作行为**
   - 查看券码 → 跳 `pages/ticket-detail/ticket-detail`。
   - 申请退票 → 先弹确认框，再调退票接口，成功后更新状态为「已退款」；三方票需提示「至原购买平台申请退款」。
   - 删除票券 → 弹出删除确认弹窗，成功后从列表移除。

 ### 3.4 互动路径

 参考 `互动.html`：

 - 上方为互动 Banner / 入口卡片（如「反应力挑战」游戏卡片）。
 - 中部为排行榜：
   - 展示前三名（名次 + 昵称 + 头像 + 分数）。
   - 显示当前用户分数与排名（若未上榜则文案「暂未上榜」）。
 - 底部为提示文案「更多游戏接入中，敬请期待！」。
 - 当前版本可以：
   - 游戏入口点击后提示「即将上线」，或进入一个简单 Demo 游戏页。
   - 排行榜先用静态 mock 数据，结构与 Axure 文案一致，后期再接接口。

 ### 3.5 我的路径

 参考 `我的.html`：

 - **头部信息**
   - 圆形头像（从微信获取）。
   - 昵称 `_weixin_xxx`，点击可进入编辑个人信息页。
 - **功能入口**
   - 我的订单：跳票夹/订单页。
   - 隐私政策 / 服务协议：跳转长文页面。
   - 联系客服：展示客服电话（如「010-88888888」），支持点击拨号。
   - 发票抬头：跳转发票抬头编辑页（填公司名、税号、地址电话、开户行等）。

 ## 4. 视觉与组件规范（对应 `uni.scss` 与通用样式）

 结合现有 `apps/mini/uni.scss`：

 - **色彩**
   - 主背景：`$cr7-black` / `$cr7-dark`。
   - 主品牌金：`$cr7-gold`（按钮、标签描边、选中态）+ `$gradient-gold`（大按钮、徽章）。
   - 辅助色：`$cr7-red`（警示）、`$cr7-success`、`$cr7-warning`。
 - **排版**
   - 正文：28rpx 为基础字号，重要标题用 `$font-lg` / `$font-xl`。
   - 标题使用「粗体 + 大写 CR7」与中文 PingFang 的组合，参考 Axure 中「CR7® LIFE 博物馆」写法。
 - **CR7 圆形徽章组件**
   - 结构建议：
     - 外层圆：直径 ~240rpx，边框使用 `$gradient-gold`，阴影 `$shadow-gold`。
     - 内层圆：直径 ~200rpx，背景为 `$gradient-dark` 或 `$gradient-hero`。
     - 中央文字：「CR7」大号白色字 + 细小副标题。
   - 在登录页居中使用，在首页和部分二级页用小号版本。
 - **按钮**
   - 主按钮统一使用 `.btn-gold`（渐变金 + 深色文字 + 圆角 48rpx）。
   - 次按钮使用 `.btn-outline-gold`。
 - **卡片**
   - 使用 `.card-dark` + `$shadow-card` 落地活动卡、票券卡、品牌卡。
 - **状态标签**
   - 使用 `.tag-active` / `.tag-countdown` / `.tag-ended` 对应进行中 / 倒计时 / 已结束。

 TabBar 的颜色与 `pages.json` 中配置保持一致：背景 `#111111`、选中金色 `#C9A84C`、未选中灰色 `#888888`。
