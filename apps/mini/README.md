# C罗"超越之境"购票小程序

基于 uniapp 开发的购票小程序前端项目，使用假数据实现核心功能。

## 项目结构

```
├── pages/                    # 页面目录
│   ├── index/               # 首页（信息流）
│   ├── login/               # 登录页
│   ├── event-detail/        # 活动详情
│   ├── ticket-purchase/     # 购票页面
│   ├── my-tickets/          # 我的票夹
│   ├── ticket-detail/       # 票券详情
│   ├── profile/             # 个人中心
│   ├── messages/            # 消息中心
│   ├── scan-ticket/         # 扫码验票（员工功能）
│   ├── game/                # 互动游戏
│   ├── vote/                # 赛事投票
│   ├── schedule/            # 赛事日程/C罗行程
│   └── brands/              # 联名品牌
├── utils/                   # 工具类
│   ├── mockData.js         # 假数据
│   └── storage.js          # 本地存储工具
├── static/                  # 静态资源
├── App.vue                  # 应用配置
├── main.js                  # 入口文件
├── pages.json              # 页面配置
├── manifest.json           # 应用配置
└── uni.scss                # 全局样式

```

## 运行说明

### 1. 安装依赖

项目使用 HBuilderX 开发，无需额外安装依赖。

### 2. 运行项目

#### 微信小程序
1. 在 HBuilderX 中打开项目
2. 点击"运行" -> "运行到小程序模拟器" -> "微信开发者工具"
3. 首次运行需要配置微信开发者工具路径

#### H5
1. 点击"运行" -> "运行到浏览器" -> "Chrome"

#### App
1. 点击"运行" -> "运行到手机或模拟器"

### 3. 登录测试

- 点击"微信一键登录"
- 勾选协议后登录
- 系统会随机分配普通用户或员工身份
- 员工账号会显示"扫码验票"功能

## 假数据说明

所有数据都在 `utils/mockData.js` 中定义：

- **用户数据**: mockUser, mockEmployee
- **首页卡片**: mockHomeCards
- **赛事数据**: mockMatches, mockStandings
- **票券数据**: mockTicketTypes, mockMyTickets
- **消息数据**: mockMessages
- **投票数据**: mockVoteOptions
- **游戏排行**: mockGameRanking
- **品牌数据**: mockBrands
- **行程数据**: mockSchedule, mockCareer

1. 浏览活动 → 2. 查看详情 → 3. 选择票种 → 4. 选择日期 → 5. 确认支付 → 6. 查看票券

- 开展前48小时可全额退款
- 早鸟票不支持退票
- 已使用票券不可退票

## 注意事项

1. 本项目使用假数据，所有操作都是模拟的
2. 支付功能仅为演示，不会产生真实交易
3. 扫码验票功能需要在真机环境测试
4. 小程序跳转功能需要配置真实的 appId
5. 图片资源使用占位图，实际项目需替换真实图片
